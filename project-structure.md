# Структура проекта административного сервиса учета проектов и технологий

## Общая структура каталогов

```
src/
├── projects/
│   ├── domain/          // Бизнес-логика и сущности проектов
│   ├── application/      // Сервисы и use-case'ы управления проектами
│   ├── infrastructure/  // Репозитории, DTO, внешние интерфейсы
│   └── interfaces/      // REST/GraphQL контроллеры
│
├── technologies/
│   ├── domain/          // Бизнес-логика и сущности технологий
│   ├── application/     // Сервисы управления технологическим стеком
│   ├── infrastructure/ // Репозитории, DTO
│   └── interfaces/      // API контроллеры
│
├── tasks/
│   ├── domain/          // Сущности и логика задач
│   ├── application/     // Сервисы управления задачами
│   ├── infrastructure/  // Репозитории, DTO
│   └── interfaces/     // API контроллеры
│
├── risks/
│   ├── domain/          // Сущности и логика управления рисками
│   ├── application/     // Сервисы оценки и контроля рисков
│   ├── infrastructure/  // Репозитории, DTO
│   └── interfaces/     // API контроллеры
│
├── kpis/
│   ├── domain/          // Сущности KPI и метрик
│   ├── application/     // Сервисы расчета и анализа KPI
│   ├── infrastructure/  // Репозитории, DTO
│   └── interfaces/       // API контроллеры
│
├── auth/
│   ├── domain/          // Сущности пользователей и ролей
│   ├── application/      // Сервисы аутентификации и авторизации
│   ├── infrastructure/ // Репозитории, DTO
│   └── interfaces/     // API контроллеры
│
├── shared/
│   ├── exceptions/     // Общие исключения
│   ├── types/          // Общие типы и интерфейсы
│   └── utils/          // Вспомогательные функции
│
├── config/             // Конфигурационные файлы
├── database/          // Схемы и миграции базы данных
└── main.ts            // Точка входа приложения
```

## Основные модули системы

### 1. Модуль управления проектами (projects/)
Отвечает за создание, управление и отслеживание проектов:
- Создание и редактирование проектов
- Управление статусами и фазами
- Расчет бюджета и прогнозов
- Управление командой проекта

### 2. Модуль управления технологиями (technologies/)
Управляет технологическим стеком:
- Каталог используемых технологий
- Отслеживание жизненного цикла
- Предупреждения об устаревании

### 3. Модуль управления задачами (tasks/)
Планирование и контроль выполнения задач:
- Создание и управление задачами
- Отслеживание зависимостей
- Контроль сроков выполнения

### 4. Модуль управления рисками (risks/)
Идентификация и контроль рисков:
- Регистрация рисков
- Оценка вероятности и влияния
- Планирование мер по снижению

### 5. Модуль KPI и отчетности (kpis/)
Аналитика и отчетность:
- Сбор и расчет KPI
- Формирование отчетов
- Экспорт данных

### 6. Модуль аутентификации и авторизации (auth/)
Безопасность и контроль доступа:
- Управление пользователями
- Ролевая система доступа
- Аудит действий пользователей

## Общие компоненты (shared/)
Предоставляют переиспользуемые элементы:
- Общие типы данных
- Утилиты и вспомогательные функции
- Обработка исключений

## Конфигурация и база данных
- Конфигурационные файлы приложения
- Схемы и миграции базы данных
